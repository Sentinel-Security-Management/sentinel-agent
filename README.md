## Contributor Rewards Program

Sentinel values meaningful contributions from the community.

At the discretion of the maintainers, contributors may receive rewards based on the quality, impact, and long-term value of their contributions to the project.

Potential rewards may include:

* Early access to new features
* Public recognition in release notes
* Contributor badges and acknowledgements
* Free or discounted access to Sentinel Cloud
* Free Enterprise-tier credits
* Invitations to beta programs
* Opportunities to become project maintainers

### Evaluation Criteria

Contributions may be evaluated based on:

* Technical quality
* Security impact
* Performance improvements
* Documentation quality
* Community impact
* Feature completeness
* Long-term maintainability
* Alignment with project goals

Examples of eligible contributions include:

* Significant features
* Security improvements
* Major bug fixes
* Performance optimizations
* Documentation improvements
* Testing infrastructure
* Developer experience improvements

### Important Notice

Rewards are granted solely at the discretion of "The Deployment Lab".

Submitting a contribution does not guarantee eligibility for any reward, credit, compensation, or future benefit.

Reward programs, eligibility criteria, and available benefits may change over time.

### Release-Based Conversion Policy

Sentinel Agent follows a release-based licensing model.

Each official tagged release is licensed under FSL-1.1-ALv2 upon publication.

Exactly twenty-four (24) months after the publication date of a tagged release, that specific release automatically becomes available under the Apache License 2.0.

Examples:

* v0.1.0 released on 2026-06-04 → Apache 2.0 on 2028-06-04
* v0.2.0 released on 2026-09-01 → Apache 2.0 on 2028-09-01
* v1.0.0 released on 2028-01-15 → Apache 2.0 on 2030-01-15
